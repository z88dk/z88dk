## z80asm File formats (v18)
[[Top](Tool---z80asm)]

This document describes the object and libary formats used by *z80asm*. 

The object and library files are stored in binary form as a set of 
contiguous sections, i.e. each section follows the previous one. 

The files start with a signature that includes a version number and is 
verified by the assembler before trying to parse the file.

A set of file pointers at the start of the object file point to the 
start of each section existing the in the file, or contain *0xFFFFFFFF* 
(-1) if that section does not exist.

The following object types exist in the file:

* *char* :	one signed 8-bit value containing an ASCII character;

* *byte* :	one unsigned 8-bit value;

* *long* :	one signed 32-bit value, stored in low byte - high byte order;

* *strid* : one signed 32-bit value indicating the string index in the 
string table; index 0 allways points to an empty string;

Object file format
------------------

The format of the object file is as follows:

|Addr  |Type   |Object|
|---   |---    |---   |
|&nbsp;|&nbsp; | **File header**|
|   0  |8 chars|'Z80RMF18' - file signature and version|
|   8  | long  | CPU id: <br> 1 : z80 <br> 2 : z80_strict <br> 3 : z180 <br> 4 : ez80_z80 <br> 5 : ez80 <br> 6 : z80n <br> 7 : r2ka <br> 8 : r3k <br> 9 : gbz80 <br> 10 : i8080 <br> 11 : i8085 <br> 12 : r800 <br> 13 : r4k <br> 14 : r5k <br> 15 : kc160 <br> 16 : kc160_z80 <br> 17 : i8080_strict <br> 18 : i8085_strict <br> 19 : gbz80_strict <br> 20 : z180_strict <br> 21 : z80n_strict <br> 22 : ez80_z80_strict <br> 23 : ez80_strict <br> 24 : r800_strict <br> 25 : kc160_strict <br> 26 : kc160_z80_strict <br> 27 : r2ka_strict <br> 28 : r3k_strict <br> 29 : r4k_strict <br> 30 : r5k_strict <br> 31 : r6k <br> 32 : r6k_strict <br> 33 : ti83 <br> 34 : ti83_strict <br> 35 : ti83plus <br> 36 : ti83plus_strict <br> |
|  12  | long  | -IXIY option used in the assembly: <br> 0 : no -IXIY option used <br> 1 : -IXIY option used |
|  16  | long  | File pointer to *Module Name* |
|  20  | long  | File pointer to *Expressions*, may be -1               |
|  24  | long  | File pointer to *Defined Symbols*, may be -1              |
|  28  | long  | File pointer to *External Symbols*, may be -1            |
|  32  | long  | File pointer to *Sections*, may be -1              |
|  36  | long  | File pointer to *String Table*                         |
|&nbsp;|&nbsp; | **Expressions**<br> expressions defined in this module
|expr+0| long  | Expression type: <br> 0 : end marker <br> 1 : signed 8-bit value, JR-offset <br> 2 : unsigned 8-bit value <br> 3 : signed 8-bit value <br> 4 : 16-bit value <br> 5 : 16-bit value, big-endian <br> 6 : 32-bit value <br> 7 : unsigned 8-bit value, extended to 16 bits (appending a 0) <br> 8 : signed 8-bit value, sign-extended to 16 bits (appending a 0xff if negative or 0 otherwise) <br> 9 : 24-bit value <br> 10 : unsigned 8-bit value, offset to 0xFF00 <br> 11 : DEFC expression copmputed at link time and assigning a symbol <br> 12 : signed 16-bit value, JRE-offset |
|&nbsp;| strid | Source file name where expression was defined |
|&nbsp;| long  | Line number in source file where expression was defined |
|&nbsp;| strid | Section name where expression was defined |
|&nbsp;| long  | *ASMPC*: relative module code address of the start of the assembly instruction to be used as *ASMPC* during expression evaluation |
|&nbsp;| long  | *patchptr*: relative module code patch pointer to store the result of evaluating the expression |
|&nbsp;| long  | *opcode_size*: size of the instruction using this expression, needed to compute relative jumps |
|&nbsp;| strid | *target_name*: name of the symbol that receives the result of evaluating the expression, only used for type 11 expressions, empty string for the other types |
|&nbsp;| strid | *expression*: expression text as parsed from the source file |
|...|...|...|
|&nbsp; |&nbsp; | **Defined Symbols**<br>symbols defined in this module |
|names+0| long  | *scope* of the name: <br> 0 : end marker <br> 1 : is local <br> 2 : is public |
|&nbsp; | long  | *type*: defines whether it is a: <br> 1 : a constant <br> 2 : a relocatable address <br> 3 : a symbol computed at link time, the corresponding expression is in the *Expressions* section |
|&nbsp; | strid | Section name where symbol was defined |
|&nbsp; | long  | *value*: contains the absolute value for a constant, or the relative address to the start of the code block for a relocatable address |
|&nbsp; | strid | *name*: contains the name |
|&nbsp; | strid | *filename*: contains the source file name where the symbol was defined |
|&nbsp; | long  | *line_nr*: contains the source line number where the symbols was defined |
|...|...|...|
|&nbsp; |&nbsp; | **External Symbols**<br> external symbols names referred by this module |
|names+0| strid | *name*: external symbol name, empty string to signal last
|...|...|...|
|&nbsp; |&nbsp; | **Module Name** <br> module name
|name+0 | strid | *name*: module name
|...|...|...|
|&nbsp; |&nbsp; | **Sections**<br>contains the binary code of each section of the module |
|code+0 | long  | *length*: defines the total code length, -1 to signal the end |
|&nbsp; | strid | *section*: source file section name
|&nbsp; | long  | *ORG Address*: contains the user defined ORG address for the start of this section,<br>-1 for no ORG address was defined, or<br>-2 to split section to a different binary file<br>If multiple sections are given with an ORG address each, the assembler generates one binary file for each section with a defined ORG, followed by all sections without one |
|&nbsp; | long  | *ALIGN*: defines the address alignment of this section,<br>-1 if not defined<br>The previous section is padded to align the start address of this section |
|&nbsp; |byte[length]| *code*: contains the binary code |
|...|...|...|
|&nbsp; |&nbsp; | **String Table**<br>contains the array of strings used in the object file |
|table+0| long  | *count*, number of strings in the table |
|&nbsp; | long  | *size*, total size of blob of all the strings rounded up to the next aligned address |
|...|...|...|
|...|...| Table of indices: |
|&nbsp; | long  | *index*, index of first character of this string in the end blob of all strings |
|...|...|...|
|...|...| Blob of strings: |
|&nbsp; | char[] | Characters of the string followed by '\0' terminator |

Library file format
-------------------

The library file format is a sequence of object files with additional
structures.

|Addr  |Type   |Object|
|---   |---    |---   |
|&nbsp;|&nbsp; |**File header**
|   0  |8 chars|'Z80LMF18' - file signature and version
|   8  | long  | File pointer to *String Table*                         |
|&nbsp;|&nbsp; |**Object Modules**<br>object files in the library
|mod+0 | long  |*next*: contains the file pointer of the next object file in the library, or<br>*-1* if this object file is the last in the library
|&nbsp;| long  |*length*: contains the length of this object file, or<br>*0* if this object files has been marked "deleted" and will not be used
|&nbsp;|byte[length]|contains the object module
|...|...|...|
|&nbsp; |&nbsp; | **String Table**<br>contains the array of PUBLIC symbols defined in any of the object files |
|table+0| long  | *count*, number of symbols in the table |
|&nbsp; | long  | *size*, total size of blob of all the symbols rounded up to the next aligned address |
|...|...|...|
|...|...| Table of indices: |
|&nbsp; | long  | *index*, index of first character of this symbol in the end blob of all strings |
|...|...|...|
|...|...| Blob of strings: |
|&nbsp; | char[] | Characters of the symbol followed by '\0' terminator |
History
-------

|Version|Description
|---    |---
|*01*   |original z80asm version
|*02*   |allow expressions to use standard C operators instead of the original (legacy) z80asm specific syntax
|*03*   |include the address of the start of the assembly instruction in the object file, so that expressions with ASMPC are correctly computed at link time;<br>remove type 'X' symbols (global library), no longer used
|*04*   |include the source file location of expressions in order to give meaningful link-time error messages
|*05*   |include source code sections
|*06*   |incomplete implementation, fixed in version *07*
|*07*   |include *DEFC* symbols that are defined as an expression using other symbols and are computed at link time, after all addresses are allocated
|*08*   |include a user defined ORG address per section
|*09*   |include the file and line number where each symbol was defined
|*10*   |allow a section alignment to be defined
|*11*   |allow big-endian 16-bit expressions to be patched; these big-endian values are used in the ZXN coper unit
|*12*   |allow the target expression of relative jumps to be computed in the link phase
|*13*   |add 8-bit signed and unsigned values extended to 16-bits
|*14*   |add 24-bit pointers
|*15*   |add byte offset to 0xFF00
|*16*   |convert all *string* to *lstring* in the object file to allow for very large symbol names, needed for C debugging symbols
|*17*   |add size of opcode so that DJNZ emulation on Intel and ALD DJNZ on Rabbit compute the right offset
|*18*   |add CPU and -IXIY option to disallow linking of objects built for a different architecture; add a string table to remove duplicate strings and speed up linking
