Linker script

-T memmap.ld

BANK <name>
    BASE <addr>
    SIZE <size>

    HOLE    <name> AT <addr> SIZE <size>    ; hardware hole
    RESERVE <name> AT <addr> SIZE <size>    ; reserve memory

GROUP CODE = code_*
GROUP DATA = data_*
GROUP BSS  = bss_*

PLACE CODE IN BANK0
PLACE DATA AFTER CODE
PLACE BSS  IN BANK1

bank switching


BANK COMMON 0x0000 SIZE 16K

BANK GAME0  0x4000 SIZE 16K
BANK GAME1  0x4000 SIZE 16K
BANK GAME2  0x4000 SIZE 16K

BANKSWITCH PORT 0x7F

TRAMPOLINES IN COMMON

PLACE CORE     IN COMMON
PLACE GRAPHICS IN GAME0
PLACE AUDIO    IN GAME1
PLACE AI       IN GAME2

AUTO FARCALLS


BANK COMMON 0x0000 SIZE 16K

BANK GAME0 NUMBER 0
BANK GAME1 NUMBER 1
BANK GAME2 NUMBER 2

FARCALLS USING __bank_call

Then the linker generates calls to __bank_call, passing the destination bank and address according to a documented ABI.
That keeps the linker portable across virtually every Z80 banking scheme while still providing all the value of automatic trampoline generation. In my view, that's the cleanest separation of responsibilities.



Mark and sweep algorithm:

selected_modules = input modules
unresolved = externs from input modules

repeat:
    added_any = false

    for each library in input order:
        for each module in library order:
            if module not selected:
                if module defines any symbol in unresolved:
                    select(module)
                    unresolved += module.externs
                    added_any = true

    remove from unresolved all symbols now defined in selected_modules

until added_any == false

if unresolved not empty:
    error("Undefined symbols: ...")

std::vector<ObjectModule*> final_modules;

for (auto& lib : libraries_in_input_order) {
    for (auto& mod : lib->modules) {
        if (selected_modules.contains(mod.get())) {
            final_modules.push_back(mod.get());
        }
    }
}

for (auto& mod : input_modules_in_order) {
    final_modules.push_back(mod);
}
